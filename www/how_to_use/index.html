How to Use the JTF (Just Transfer File) App?

Welcome to the JTF (Just Transfer File) application - your secure file transfer solution owned and operated by zohiddev.me. Before you begin, please read the Terms of Service carefully, as they govern your use of the JTF application.

Read below to learn how to use the JTF App:

1. Sign Up and Get Subdomain:

To access secure file transfer, sign up or sign in to your account. Upon successful registration, you will receive a unique subdomain associated with your verified account. This subdomain will be displayed on the download page of files sent from your account, indicating that the files are coming from a verified user.

2. Link Your SSH Key:

Once signed in, link your laptop's SSH key to your account. This process establishes a secure connection between your device and our platform, ensuring encrypted and safe file transfer.

3. Sending Files via SSH:

With your SSH key linked, you can securely send files from your command-line environment using the "ssh" command. For example: ssh jtf.zohiddev.me -p 2222 < file where "jtf.zohiddev.me" is our base domain, and "file" represents the file you want to send. Below you can see other style of sending file!

a. From: ssh jtf.zohiddev.me -p 2222 from="Alex" < dump.json

b. Filename: ssh jtf.zohiddev.me -p 2222 filename="just.json" < dump.json

c. Message: ssh jtf.zohiddev.me -p 2222 msg="This file is for you" < dump.json

d. All In One: ssh jtf.zohiddev.me -p 2222 filename="just.json" msg="This file is for you" from="Alex" < dump.json

4. Verified User Display:

Files sent from your account will display your verified subdomain on the download page. This verification reassures recipients that the files are originating from a trusted and secure source.

5. Secure Link Sharing:

After sending a file, the recipient will receive a secure link to download it. The link directs them to a download page with your verified subdomain displayed, reinforcing the authenticity of the source. For added security, the link is time-limited, ensuring files are accessible only for a specific duration.

6. Automatic File Deletion:

To enhance privacy, files sent via SSH are automatically deleted after the specified time limit if the recipient does not download them within that period.

7. Developer Cannot Access File Content:

Rest assured that your files' content is entirely secure during the transfer process. Our SSH to HTTP tunnel ensures that files are never stored on our servers. Once the transfer is complete, the files are automatically deleted from our system. As a result, developers or anyone else do not have access to view or retrieve the content of the files. Your data remains private and confidential throughout the entire process.
